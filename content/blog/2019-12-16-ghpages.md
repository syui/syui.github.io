現在、hugoをgithub-actionsで自動ビルドとデプロイしているのですが、デプロイが、コミットを消すやつじゃないので困ってます。

`${user}.github.io`のリポジトリは、masterにソースが置かれます。(privateなら見えないようにできる)

しかし、この前、ちょっと裏技ぽいものを発見したのでメモ。

github-actionsでpage_buildしたあと、即座にsrcをgitでrebase pushするアクションを走らせると、リポジトリのmasterにはsrcだけが置かれてるのに、ページが表示されてました。本当は、htmlが置かれないといけない。

```yml:.github/workflows/deploy.yml
name: github pages deploy

on:
  page_build:

jobs:
  build:
    runs-on: ubuntu-18.04
    steps:
    - uses: actions/checkout@v2
      with:
        branch: 'master'
        run: git checkout master
      
    - uses: ad-m/github-push-action@master
      with:
        branch: 'master'
        force: true
        github_token: ${{ secrets.GITHUB_TOKEN }}
        run: |
           git config --local user.email "action@github.com"
           git config --local user.name "GitHub Action"
           GIT_SEQUENCE_EDITOR="sed -ie '2 s/^pick /f /g'" git rebase -i --root
```

内容は、branch:srcの内容をbranch:masterに`psuh -f`するみたいな感じになってる。(やりたいことは、masterのcommitをfixupしてcommit削減)

`GIT_SEQUENCE_EDITOR="sed -ie '2 s/^pick /f /g'" git rebase -i --root`は危ないので注意。バックアップは必ずやっておくこと。ただ、思ったとおりの動作じゃない気がする。

また、何度も連続でcommitするとgh_buildされたあと、このアクションが2回連続実行されたりすることもあるので、その場合、ページは消えてしまう。正確には、masterのREADME.mdになります。

この挙動、よくわかってませんが、これはこれで結果的にいいかなと思ってるので、放置。
